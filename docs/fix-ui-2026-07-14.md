# UI 改进记录 (v3.17.7)

> 纯客户端改动，未修改服务端 SQL / RPC / 表结构。

## 改动清单

### 1. 过去日期视觉区分
- **文件**：`src/pages/CalendarView.vue`、`src/styles.css`
- **效果**：已过去的日格自动添加 `is-past` CSS 类（`dateStr < today && !other-month`）。
  - 样式：淡淡边框 + 78% 不透明度；今天即使在过去判断中也保持不淡化。
  - 纯 CSS 方案，Electron 和 Capacitor WebView 渲染一致。

### 2. 状态颜色增强
- **文件**：`src/pages/CalendarView.vue`
- **Bug 修复**：补上缺失的 `:data-status="dayData(cd.dateStr).status"` 属性绑定。
  - 这让 `styles.css` 中已有的 7 条角标配色规则立即生效：
    `--work`(绿)、`--rest`(蓝)、`--trip`(橙)、`--leave`(紫)、`--annual`(青)、`--sick`(深橙)、`--personal`(棕)。
- **增强**：新增 `STATUS_BG_MAP` 淡底色映射，用户未手动设 color 时，按状态给日格极淡背景辅助辨识（上班=淡绿、休息=淡蓝、出差=淡橙等）。用户自定义 color 优先级不变。

### 3. 好友圈三页签
- **文件**：`src/pages/SocialPage.vue`（重写）、`src/components/FriendsTab.vue`（新建）、`src/components/ProfileTab.vue`（新建）
- **问题**：原 SocialPage.vue 只渲染动态流，好友列表(`renderFriends`)和个人资料(`renderProfile`)是传统 DOM 层实现，渲染到隐藏的 `.app > #social-content` 容器——在 Vue 主流程和安卓端完全不可见。
- **修复**：SocialPage.vue 顶部新增「动态 | 好友 | 我的」Tab 切换栏，全部 Vue 化：
  - **动态 Tab**：保留原有动态流逻辑（发帖/点赞/评论/删除），移除旧的 `switchToFriends()` 跳转。
  - **好友 Tab** (`FriendsTab.vue`)：好友列表、待处理申请（同意/拒绝）、添加好友（输入 ID 搜索+发送请求）。复用 `window.getFriends/getFriendRequests/sendFriendRequest/acceptFriendRequest/rejectFriendRequest/removeFriend` 等 API。
  - **我的 Tab** (`ProfileTab.vue`)：头像/昵称/显示ID、复制 ID、修改昵称。复用 `window.getMyProfile/updateProfile` API。
- **跨端**：全部 Vue 组件 + 已有 window.* API 桥接，Electron 和安卓 WebView 行为一致。

### 4. 密码恢复 seed 清理
- 移除 `src/public/salt-seed.js` 及 `index.html` 中的引用（密码重置已完成，不应随正式版外泄硬编码 salt）。
