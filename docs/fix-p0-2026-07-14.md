# P0 严重缺陷修复记录（2026-07-14）

> 关联审查报告：`docs/code-review-2026-07-14.md`
> 范围：4 个严重缺陷（S1–S4）全部修复，`vite build` 通过。

## S1 · 编辑待办完全失效
- **文件**：`src/components/TodoItem.vue`
- **改动**：`openEdit()` 由 `todoStore.startEdit(props.todo)` 改为 `window.__openTodoModal(props.todo)`。
- **原因**：弹窗唯一打开入口是 `window.__openTodoModal`（在 `TodoModal.vue` 定义）；原写法只写 store 局部 ref，弹窗根本不开，点 ✎ 零反应。

## S2 · 待办新增/删除后列表不刷新
- **文件**：`src/components/TodoModal.vue` / `TodoItem.vue` / `TodoListApp.vue` / `TodoViewApp.vue`
- **改动**：
  - `TodoModal.confirm()` 改调 `todoStore.addTodo/updateTodo`（内部 `loadTodos()` 重拉），并补 `window.__refreshTodoView?.()`。
  - `TodoItem.toggleDone/deleteTodo` 的 `todoStore.refreshFromWindow()` 改为 `await todoStore.loadTodos()`（IPC `addTodo` 不更新 `window.allTodos`，旧写法读到的永远是陈旧数据）。
  - `TodoListApp.updateList` 与 `TodoViewApp` 的 `onceTodos`/`weeklyTodos` 过滤均加 `!t.deleted`（主进程删除是软删 `deleted:true`，否则残留）。
- **效果**：打卡页与「全部待办」页增删后立即刷新，删除项彻底消失。

## S3 · 休息日/请假日仍弹打卡提醒（桌面端）
- **文件**：`main.js`（`checkReminders`，约 479 行后）
- **改动**：函数开头读取 `store.days[todayStr].status`，若为 `rest/leave/annual/sick/personal` 直接 `return`。
- **原因**：原逻辑只比时间+是否已确认，不读当天状态；休息日点通知会写入一条"休息日已打卡"脏记录，与 Vue 隐藏休息日卡片矛盾。

## S4 · Web/Android 假更新弹窗
- **根因**：`updater.js:33` 与 `electron/api.js:316` 都把 `fetch('version.json')` 失败后的最终回退硬编码成 `'3.13.0'`；当前真实版本 `3.17.5`，远程 ≥3.13.1 即判"有新版本"，离线 Android 必弹假更新。
- **文件/改动**：
  - `vite.config.js`：新增 `define: { __APP_VERSION__: JSON.stringify(pkg.version) }`（从 `package.json` 读取）。
  - `src/electron/api.js`：顶部将 `__APP_VERSION__` 同步到 `window.__APP_VERSION__`；Web 回退 `'3.13.0'` → `window.__APP_VERSION__ || '3.17.5'`。
  - `src/updater/updater.js`：最终回退 `'3.13.0'` → `__APP_VERSION__ || '3.17.5'`。
- **验证**：`vite build` 通过；dist bundle 已无 `3.13.0`，注入 `window.__APP_VERSION__="3.17.5"`，两处回退均返回真实版本。
